**Status**: Complete

- Hypothesis document: `doc/phase_12/hypotheses/trend_following/adaptive_ma.md` (232 lines)
- Complete unit tests (6 tests)

**Key Features**:
- Kaufman's Adaptive Moving Average (KAMA) implementation
- Efficiency ratio calculation for trend strength detection
- Adaptive smoothing (fast in trending, slow in ranging)
- Optional RSI overbought filter
- Optional ADX trending filter
- Optional volume confirmation
- ATR-based trailing stop
- Partial take profit logic

**Configuration Parameters**:
- `fast_period` - Fast EMA equivalent period for KAMA (default: 10)
- `slow_period` - Slow EMA equivalent period for KAMA (default: 30)
- `price_period` - Price change period for efficiency ratio (default: 10)
- `take_profit` - Take profit percentage (default: 5.0%)
- `stop_loss` - Stop loss percentage (default: 3.0%)

**KAMA Implementation**:
- Efficiency Ratio (ER) = |price_now - price_then| / volatility
- Smoothing Constant (SC) = [ER * (fast_SC - slow_SC) + slow_SC]²
- fast_SC = 2 / (fast_period + 1)
- slow_SC = 2 / (slow_period + 1)
- KAMA = KAMA_prev + SC * (price - KAMA_prev)

- Hypothesis document: `doc/phase_12/hypotheses/trend_following/parabolic_sar.md` (245 lines)
- Complete unit tests (4 tests)

**Key Features**:
- Parabolic SAR implementation for trend following
- SAR as trailing stop that accelerates
- Entry when price crosses SAR
- Automatic trend reversal detection
- Optional SMA trend filter
- Optional RSI overbought filter
- Optional volume confirmation
- Partial take profit logic

**Configuration Parameters**:
- `af_step` - Acceleration factor step (default: 0.02)
- `af_max` - Maximum acceleration factor (default: 0.2)
- `take_profit` - Take profit percentage (default: 5.0%)
- `stop_loss` - Stop loss percentage (default: 3.0%)

**SAR Implementation**:
- SAR tracks extreme point (EP) and accelerates
- In uptrend: SAR starts below previous low, moves up with price
- In downtrend: SAR starts above previous high, moves down with price
- AF increments by `af_step` each period (up to `af_max`)
- SAR = SAR_prev + AF * (EP - SAR_prev)
- Trend reverses when price crosses SAR

## Lessons Learned

### What Worked Well
1. **Hypothesis-First Development** - Creating hypothesis documents before coding ensured clear requirements and prevented scope creep
2. **Shared Utility for Canonicalization** - Eliminated duplication and created single source of truth for strategy name mapping
3. **Comprehensive Testing** - Extensive unit tests caught edge cases early and ensured robust code
4. **Consistent Pattern Across Strategies** - Reusing the same structure made implementation faster and code more maintainable
5. **Enhanced Indicators** - ATR, ADX, and KAMA provided powerful building blocks for multiple strategies

### Areas for Improvement
1. **Symbol Handling** - The "UNKNOWN" symbol workaround should be addressed in future by updating the Strategy trait
2. **Dashboard API Limitations** - Some strategy features (like multi-level TPs) aren't fully exposed in the API
3. **Validation Gap** - Walk-forward and Monte Carlo validation should be run immediately after implementation to ensure strategies actually work
4. **Database Integration** - Should store strategy metadata and performance in database from the start

### Recommendations for Future Phases
1. **Validation First** - Run validation tests immediately after strategy implementation to catch issues early
2. **API-Driven Development** - Design API parameters before implementing strategies to ensure full feature exposure
3. **Database-First** - Store strategy metadata and results in database from day one for better tracking
4. **Symbol Enhancement** - Update Strategy trait to accept symbol explicitly, eliminating "UNKNOWN" workaround
